Single Sign-On Flow

  1. User logs in to SCL Frontend
  2. User clicks "Access LMS" card
  3. Frontend calls /api/sso/generate
  4. Backend creates SSO token & stores in database
  5. Backend redirects to Moodle with token
  6. Moodle verifies token with backend
  7. Moodle auto-creates/updates user
  8. User auto-logged into Moodle dashboard
